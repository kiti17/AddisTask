# AddisTask First Admin Setup

Use this when preparing the real owner account for pilot operations.

## Goal

The owner/admin account should be the only account that can approve providers, hide marketplace records, restore hidden tasks, and view admin data controls.

## Before Pilot

- Create the real owner account with the phone number you will keep long term.
- Confirm the account can log in.
- Confirm Admin Mode appears only for that account.
- Open Marketplace Operations.
- Review the First Admin Checklist in Marketplace Operations.
- Confirm the Launch Checklist, Data Management, Provider Approval Queue, and admin controls are visible.
- Open Account and update the password once before inviting pilot users.

## Password Safety

- Use a password that is not used on another website.
- Keep the admin password private.
- Change the admin password if it is shared during testing.
- Repeated wrong-password login attempts are temporarily limited during the pilot.
- If a trusted tester is locked out after several mistakes, wait and try again later.
- For now, do not promise automated password reset to users until SMS or email verification is added.

## If A Pilot User Forgets Their Password

Until verified password reset is built:

- Confirm the user by phone call or direct owner contact.
- Record the issue in pilot notes.
- Resetting passwords manually should be handled only by the owner/admin or database operator.
- Do not send passwords through public chat or group messages.

## Next Production Upgrade

Add verified password reset using SMS or email before opening the app beyond a controlled pilot.
