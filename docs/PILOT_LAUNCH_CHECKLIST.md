# AddisTask Pilot Launch Checklist

Use this before inviting real pilot customers or providers.

## 1. Owner Setup

- Confirm the owner/admin account can log in.
- Confirm Admin Mode is visible only to the admin.
- Open Marketplace Operations and refresh the dashboard.
- Review the First Admin Checklist.
- Review Manual Pilot Messages in Marketplace Operations.
- Open Account and confirm Update Password works for the owner account.
- Confirm repeated wrong-password attempts show a temporary wait message.
- Keep `docs/FIRST_ADMIN_SETUP.md` available for owner/admin setup notes.
- Keep `docs/PILOT_COMMUNICATION_TEMPLATES.md` ready for customer/provider invitations.

## 2. Marketplace Cleanup

- Open Data Management.
- Run Scan Data.
- Review Pilot Cleanup Review.
- Hide old demo tasks, duplicate tasks, or records that should not appear to pilot users.
- Confirm Hidden marketplace tasks and Admin history are understandable.

## 3. Provider Readiness

- Keep the pilot small: start with 2 to 5 trusted providers.
- Confirm each provider has a complete trust checklist.
- Approve only providers with clear service area, availability, contact phone, and experience details.
- Check that customer Provider Directory shows only approved trust-ready providers.

## 4. Customer Test Setup

- Create one realistic task for each service you want to test first.
- Use clear areas such as Bole, Megenagna, Piassa, CMC, Mexico, Kazanchis, or Sar Bet.
- Keep exact address details private until a provider is accepted.
- Confirm the task appears in My Posted Tasks and Browse Jobs.

## 5. Pilot Day Flow

- Keep `docs/PILOT_USER_ONBOARDING.md` open while guiding the first users.
- Keep `docs/PILOT_RUN_SHEET.md` open while observing the test.
- Use `docs/PILOT_COMMUNICATION_TEMPLATES.md` for reminders and feedback requests.
- Watch Manual Pilot Messages for send-now reminders.
- One customer posts a task.
- One provider applies.
- The customer opens Manage Task and reviews the application.
- The customer accepts the provider.
- Both users check messages.
- The customer marks payment status and completes the task.
- The customer leaves a review.
- Admin scans Data Management and checks Pilot Run in Marketplace Operations.

## 6. Observation Notes

Record these during the session:

- Where the customer hesitated.
- Where the provider hesitated.
- Which buttons or labels were unclear.
- Whether any page felt crowded.
- Whether users had to scroll too much.
- Any action that required your help.

## 7. Stop Conditions

Pause the pilot if:

- Users cannot log in reliably.
- A normal user can see admin controls.
- Customers can see unapproved providers.
- Providers can apply before approval.
- Accepted providers cannot see task messages.
- Customers cannot complete or review a task.
